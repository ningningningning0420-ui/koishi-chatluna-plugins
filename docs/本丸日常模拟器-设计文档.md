# 本丸日常模拟器（chatluna-life-sim）设计文档 · 定稿

> 状态：设计定稿（不改运行代码）
> 日期：2026-06-30
> 作者：妖祀 / Claude
> 一句话：把每个刀剑男士做成一个 **multi-agent**（参照斯坦福小镇 Generative Agents），在没人聊天时自己「过日子」——观察→记忆→反思→规划→行动的循环，长出一段**持续、一致、可被引用**的不在场生活，于是「不聊天，世界也在长」。

---

## 0. TL;DR

- **架构**：multi-agent。每个刀剑男士 = 一个 agent，各自跑斯坦福小镇式循环（观察 Observe → 记忆流 Memory → 反思 Reflect → 规划 Plan → 行动 Act）。agent 之间由「共享线索/日程/关系 + 近侍协调」触发交互（§3）。
- **记忆分两层，全程直取，不靠向量彩票**：短期（工作集，按 recency 直接注入）+ 长期（按 key 直取，relevance 向量仅作可选联想）。斯坦福的 recency/importance/relevance 三信号在这里**各司其职**（§4）。
- **与 livingmemory 分域协同**：livingmemory 管「和用户对话」的记忆（不动）；sim-memory 管「不在场生活」的记忆（自成两层、自己注入、不写它向量）。沉淀逻辑复用 livingmemory 的 dream 思路，各库各跑（§4.6）。
- **它比"模型幻觉"多的唯一硬价值**：**跨时间一致 + 闭合因果**。幻觉每次编得都不一样、无法对质；模拟器记的是同一段、你能引用的历史（"你不是说在读那本书？"对得上）。要实现它，恰恰**不能靠向量检索低重要度日常**，而要靠直取式短期工作集 + 按 key 直取的长期档案（§1、§4）。
- **行动受约束**：JSON schema 框结构 + life-state 锚 + 每次重注人设 canon（查 worldbook）+ 计划当主干 → 让「半自由叙事」既生动又不崩人设。世界有**时空连续性**（时钟单调/移动花时间/天气相邻转移）；出本丸走**许可制 + 外部世界书**（§5）。
- **主动外联**：无 importance 门槛，发不发/何时发/没回是否再发全交人设判断；只留两道**可开关**硬熔断（安静时段 + 每日上限）（§5）。
- **MVP**：P1 单 agent 自循环（短期记忆 + 夜间沉淀 + 自著日程 + 最小主动分享）；P2 反思/长期压缩/周报/突发·节日；P3 多 agent（ChatProxy + 关系档案 + 近侍协调）（§9）。

## 1. 价值判断

### 1.1 它真正解决什么，以及"为什么不是模型幻觉就够了"

当前 bot 的连续生活感完全依赖被动对话——不聊就不长。但**关键的反问是：你问"最近在干嘛"，模型张口就能编，凭什么要建这套东西？**

诚实的答案：**只要你从不回头核对，幻觉确实够用。** 模拟器唯一、且幻觉根本做不到的价值是——

- **跨时间一致性**：幻觉每次给的答案都不同（今天说在读兵法书，明天问"读完没"就露馅）。模拟器记的是**同一段、持续、你能拿来对质**的历史。
- **闭合因果**：「约了明天对练」→ 明天真发生 → 后天你提起对得上。幻觉没有持久状态，每次提及都是孤立的，闭不了环。
- **跨 agent 一致**：髭切和膝丸对同一段共同经历记得一致。两个独立模型上下文靠幻觉无法协调。

### 1.2 这决定了一条硬设计约束：别靠向量捞低价值日常

日常生活记忆**重要度天然低、语义又不一定贴近当前话题**——正是 embedding + importance 排序检索最容易埋掉、抓不到的那类。所以"归档进向量库等它自然浮现"是错的路。**正确路径是直取**：最近的按 recency 直接注入（永远在场），关系/线索按 key 直取。低重要度不再是问题，因为我们不拿重要度来 gate 日常注入（§4）。

### 1.3 有没有更简单的替代

| 替代 | 成本 | 一致性 | 结论 |
|---|---|---|---|
| 纯模型幻觉（问就编） | 0 | **无**（每次不一样） | 你若从不对质，够用；要连续/可引用则不行 |
| 纯随机事件池（不调模型抽签） | 0 | 有一致但**不长、重样** | 当冷启动/降级兜底 |
| **本设计（agent 循环 + 直取式长短期记忆）** | 低（便宜模型/本地免费） | **有，且可引用** | 推荐 |

→ 纯随机池保留为可切换的**兜底挡位**；默认走 agent 循环。

## 3. 架构：multi-agent（斯坦福小镇映射）

### 3.1 每个刀剑男士 = 一个 agent，跑同一个循环

```
                    ┌──────────── 一个刀剑男士 = 一个 Agent ────────────┐
  闲置>idle(30min)  │ ① 观察 Observe: roll事件/WorldContext变化/遇到别的agent/用户消息 │
  (持久化调度触发)   │        ▼                                             │
                    │ ② 记忆流 Memory: 短期(工作集,recency直取) ⇄ 长期(key直取+可选联想) │
                    │        ▼                                             │
                    │ ③ 反思 Reflect: 累积重要度/夜间 → 关系·自我认知 → 写回长期  │
                    │        ▼                                             │
                    │ ④ 规划 Plan: 自著常规日程 + 被安排 → 每日计划(主干) → 打断重规划 │
                    │        ▼                                             │
                    │ ⑤ 行动 Act: 便宜模型 roll 一个行动/事件(计划主干,list-then-roll)│
                    │        │ 输出: event + plan_adherence + want_to_share + next_state + next_delay │
                    └────────┼────────────────────────┬─────────────────────┘
                             │(行动涉及别的 agent)      │(直接注入，非向量)
              ┌──────────────▼───────────┐   主聊注入: {recent_life}{life_state}{today_plan}
   [近侍/orchestrator] 协调谁与谁交互       │   {relationship:X}{pending_thoughts} + 既有{living_memory}
              │ [ChatProxy] 便宜模型多轮互聊 → transcript + 每对"关系档案" + 双方各若干条记忆
              └──────────────────────────┘
  [审神者/用户]: 终审日程/介入/可选围观(visibleVia:relay)
  [WorldContext]: 本丸级 时钟/天气(自演化)/季节/地点，跨 agent 共享
  [持久化 Scheduler]: DB 落库 + 启动扫描，重启不失忆
```

### 3.2 斯坦福机制 → 本丸映射

| Stanford Generative Agents | 本丸落点 |
|---|---|
| Agent（小镇居民） | 一个刀剑男士 preset = 一个 agent |
| Memory stream（观察为 NL + 时间戳） | 短期记忆流：roll 的事件/行动/观察/交互/用户消息（§4.1） |
| Observation | roll 出的事件、WorldContext 变化、遇到别的 agent、收到用户消息 |
| Reflection（重要度和>150 → 3 问 → 每问 5 洞见，写回流、成树） | 反思模块：累积重要度或夜间触发 → 合成关系认知/自我认知 → 写回长期（§5.4） |
| Planning（日→时→5-15min，反应即从该点重规划） | 自著常规日程 + 被安排 → 每日计划(主干) → 打断重规划（§5.3） |
| Retrieval（recency+importance+relevance 等权归一） | **三信号拆开各司其职**：recency→短期工作集直取；importance→沉淀/遗忘；relevance→可选联想深取（§4.4） |
| Agent 对话（共处→取对对方的记忆+意图→轮替至一方结束→记回流） | ChatProxy：近侍/orchestrator 触发 → 便宜模型多轮 → transcript + 关系档案 + 双方记忆（§3.3） |
| 空间世界 / 共处判定 | 本丸无坐标 → 用「共享线索/日程/关系 + 近侍协调」决定谁与谁交互（§3.3） |
| Agent summary / identity seed | 人设卡 + worldbook canon（§5.5） |

### 3.3 agent 间交互：近侍协调 + ChatProxy（默认内部模拟）

斯坦福靠**空间共处**触发对话；本丸无坐标，所以交互由**协调层**决定：

- **触发**：两 agent 有共享线索（约定/关系里程碑/共同日程块）、或近侍编排了共同活动时，发起一次交互。
- **ChatProxy（默认，内部模拟，不发真消息）**：便宜模型 + 双方精简人设卡跑多轮真对话 → 产出：
  1. 逐句 **transcript**（原始事实源，冷存，需要深挖细节时才翻）；
  2. 每对 **关系档案**（按 (botA,botB) 直取的共同经历摘要 + 未了约定 + 关系基调，§4.2）；
  3. 双方**各若干条**第一人称记忆（条数按对话内容定，同源 transcript → 事实对齐、心情按人设分化）。
- **为什么默认不走 relay**：relay 每轮要各自主模型 + 完整预设窗口，又贵又慢；且你**没有围观需求，只要记忆一致 + 需要时 rich context**。ChatProxy 便宜、可控、产物正好是直取式共享记忆。
- **relay 仅作"围观升级"**：想偶尔看两个本丸刀真聊，配 `visibleVia: relay`（此时才真发消息、真跑主模型，可回放 transcript）。
- **防跑飞**：轮数帽 `botChatMaxTurns` + `TERMINATE` 语义收尾 + 近期消息哈希防循环 + 每轮末尾重注角色卡防 echoing（文献 15–25% 同化率）；便宜模型 + 精简卡使 K 可较大；全局 `botChatDailyCap`。

## 4. 记忆系统：长短期两层（核心）

### 4.1 短期记忆 STM（工作集）

- **存什么**：最近的观察/事件、当天日程进度、刚发生的交互、当前 mood/state。
- **表示**：全量逐条事件（`life_sim_event`）+ life-state 对象（§5.2）+ today-plan + 逐句 transcript（若有交互）。
- **生命周期**：滑动窗口（最近 `stmDays` 天 / `stmMax` 条）；旧项在夜间沉淀进长期后滚出丢弃。
- **怎么取**：按 **recency 直取**，**永远在场、直接注入**（`{recent_life}`/`{life_state}`/`{today_plan}`）——不评分、不向量。这覆盖"日常连续性"，正是向量最容易漏的部分。

`life_sim_event`（当天/近期缓存）：
```
id, presetId, day, ts, title, narrative, event_type, location, participants(json),
mood, duration_min, importance(0-1), threads(json), plan_adherence, type, consolidated(bool), sourceModel
```

### 4.2 长期记忆 LTM

- **存什么**：沉淀后的重要事件、**每对关系档案**、未了线/习惯、反思洞见、自我认知。
- **表示**：蒸馏条目（带 importance/keywords/entity-key）+ 关系档案 + 反思节点。
- **生命周期**：由短→长沉淀生成（§4.3）；长期内部再 **dream 式压缩**（keep/update/merge/archive）；按 importance + 引用计数遗忘（§4.5）。
- **怎么取**：主按 **key 直取**——`relationship(a,b)` / `thread(id)` / `entity(name)` / `date(周年)`；**relevance 向量仅作可选联想深取**（给长期库建 embedding 当附加索引，非骨干）。

`life_sim_ltm`（长期条目）：
```
id, presetId, kind(event|insight|habit), content, summary, keywords(json),
entities(json), importance, refCount, createdAt, lastAccessedAt, embedding?(可选)
```
`life_sim_relationship`（每对关系档案，直取）：
```
pairKey('髭切|膝丸'), presetA, presetB, summary(滚动摘要), openThreads(json),
tone(关系基调), lastChatId, updatedAt
```

### 4.3 短→长沉淀（consolidation，复用 livingmemory dream 思路）

- **时机**：夜间「睡觉」（`sleepHour`）+ 反思触发（累积重要度过阈，§5.4）。
- **过程**（作用于 sim 自己的库，算法family 同 livingmemory dream）：
  1. 取当天短期事件流 + 交互 transcript；
  2. 便宜/主模型**聚类 + 决策 keep/update/merge/archive**（按 keywords/entity/时间聚类，importance 加权）；
  3. 生成/更新**长期条目**、**关系档案**（滚动摘要 + 未了线）、**反思洞见**；
  4. 短期窗口旧项滚出丢弃（已蒸馏进长期）。
- **风格对齐**：长期 `content` 用第一人称、≤100 字、keywords≤12、不堆日期——和 livingmemory 条目同构，便于将来互操作。

### 4.4 检索：斯坦福三信号各司其职（"为什么不靠向量彩票"的正解）

斯坦福把 `recency+importance+relevance` 加权用于**每一次**检索。我们**拆开分工**：

- **recency** → 决定**短期工作集**：最近的永远直接注入，无需检索。→ 覆盖日常连续性。
- **importance** → 决定**谁沉淀进长期、谁被遗忘**（生命周期），**不**拿来 gate 日常注入。
- **relevance（向量）** → 只在需要"联想很久以前的旧事"时，对**长期库**做**可选**深取；不是日常主干。

→ 低重要度日常永远靠 recency 直取覆盖，绝不进彩票；向量只服务稀有的深度联想。这条是整套记忆设计的定盘星。

### 4.5 遗忘 / 压缩（sim 自己的"dream"，直取式）

- 短期：滑动窗口，旧的沉淀后滚出。
- 长期条目：dream 式聚合（merge/archive）；遗忘按 `importance × 引用计数`（被反复触及的 thread/关系不删，参考 Letta reference-count 优先）。
- 关系档案：定期 LLM 重新滚动摘要 + 未了线（解决即关）。
- transcript：近的留逐句，旧的压缩/归档（关系档案已留蒸馏版）。
- 每周 `WeeklyPruner`：清 `importance < pruneThreshold` 且已沉淀的短期残留。

### 4.6 与 livingmemory 的边界与协同

| | livingmemory | sim-memory |
|---|---|---|
| 域 | 和**用户对话**的记忆 | bot **不在场生活**的记忆 |
| 检索 | vector + importance（**不动**） | 直取为主 + 可选联想 |
| 注入变量 | `{living_memory}` | `{recent_life}/{life_state}/{today_plan}/{relationship}/{pending_thoughts}` |
| 整理 | 自带 dream | 复用同思路，各库各跑 |

**四个协同点**：
1. **并排注入互补**：主聊 prompt 里 livingmemory 管"聊过啥"、sim-memory 管"私下经历啥"，互不污染。
2. **整理逻辑一致**：sim 沉淀复用 livingmemory 的 dream 算法family（keep/update/merge/archive + 重要度聚类），保证两套系统"整理思路统一"，但**各管各的库**。
3. **自然回流**：某生活事件**真被拿到对话里聊** → 那次对话经正常 chat pipeline **自然**成为 livingmemory 条目。重要生活事件因此以自然方式进入"关系记忆"，**sim 不硬写 livingmemory**。
4. **可选联想**：将来可给 sim 长期库建 embedding，复用 livingmemory 同款 retriever 做附加联想深取。
- **硬边界**：sim 绝不把生活事件写进 livingmemory 的向量库；两域数据独立、可各自清理。

## 5. 行动循环细节（Act / Plan / World / 外联）

### 5.1 闲置触发 & 行动 roll

- **信号源**：`message_collect` → `lastActivity[presetId]=now`。不轮询；每次 roll 完若距 lastActivity < `idleThreshold`（默认 30min）则把下次 roll 推到 `lastActivity+idleThreshold`（人来了让世界让位）。落在 `quietHours`（默认 22:00–08:00）内的推到醒来。
- **行动 roll 输入**（静态在前，利于 prompt cache）：
  1. 角色 canon：人设摘要 + worldbook 查询（`selectEntries` 拿相关条目）
  2. `life-state`（§5.2）
  3. WorldContext（时段/天气/季节/地点，§5.4world）
  4. **当前日程块**：DailyPlan 里此刻计划在做的 activity/location（§5.3）
  5. **当前可发生事件类型集**：EventRegistry 按 WorldContext 筛出
  6. **短期记忆**：`recent(presetId,n)` 直取最近若干条
  7. **相关长期记忆**：按当前 thread/entity **key 直取**（非向量）
  8. 主动外联沉默状态（未回时长/连续未回数）→ 供 want_to_share
  9. 指令：先列 3–5 候选 beat（list-then-roll），程序端随机抽 1
- **行动 roll 输出（JSON schema 约束）**：
```jsonc
{
  "candidates": ["去库房清点刀装","和膝丸在檐下喝茶","练习时被本丸的猫绊了一下", ...],
  "chosen_index": 2,             // 程序端最终抽样，可按 thread 相关度加权
  "event": {
    "title": "练习时被猫绊了一下", "narrative": "≤150字第一人称 beat",
    "event_type": "角色互动",      // 取自当前可发生类型集
    "location": "本丸·练习场", "participants": ["本丸的猫"],
    "mood": "好笑/无奈", "duration_minutes": 40,
    "importance": 0.3,            // 仅用于沉淀/遗忘排序，不做外联门槛
    "threads_touched": ["与猫的日常"], "type": "context"
  },
  "plan_adherence": "deviated",   // followed | deviated | interrupted | free
  "replan_hint": "猫把练习搅了，这块提前收",
  "want_to_share": {              // 人设主导（§5.7），折进本次调用，零额外成本
    "decision": "no",             // now | later | no
    "target": "审神者",
    "reason": "这点小事不值当特意说",
    "draft": "",                  // now 时草稿（分享真事，禁挽留/情感勒索）
    "thought": "下次见面提一句野花"  // later 时 → 入 ThoughtBuffer（不排闹钟）
  },
  "next_state": { /* 更新后 life-state */ },
  "next_delay_minutes": 55
}
```
- **护栏**（§6.3 详）：schema 框结构；narrative 仅单 beat 自由；不违反 worldbook 硬事实；`event_type` 取自动态可发生集；不落硬禁止清单（新命名角色/重大伤亡）；出本丸走许可制（§5.4d）。`fallbackToTemplate` 随机池兜底。

### 5.2 life-state 状态对象（防漂移锚）

始终回喂便宜模型的小状态块，存 `life_sim_state`：
```jsonc
{
  "presetId": "髭切-通用版（Character）",
  "location": "本丸·檐下", "current_activity": "擦拭刀身",
  "mood": "慵懒",                 // 随事件波动（额外功能 §7）
  "open_threads": [ {"id":"膝丸约练","desc":"答应明天对练","due":"明日"} ],
  "recent_event_ids": ["evt_...","evt_..."],
  "updatedAt": 1719...
}
```
每次 roll 后由 `next_state` 覆盖更新；同时注入对话（`{life_state}`）让 bot 知道"此刻在做什么"。

### 5.3 日程：自著常规日程 + 被安排 + 每日计划 + 治理三层级

**(a) 常规日程 RoutineTemplate（"每天常规做什么"）—— bot 自己主设计**：
```jsonc
{ "presetId":"...", "authoredBy":"self", "revisedAt":1719...,
  "weekly": { "default": [
    {"block":"清晨","activity":"起身、擦拭刀身","location":"自室"},
    {"block":"上午","activity":"练习场对练","location":"练习场"},
    {"block":"午后","activity":"巡视本丸 / 看书","location":"庭院"},
    {"block":"黄昏","activity":"泡茶、发呆","location":"檐下"},
    {"block":"夜","activity":"夜巡","location":"本丸各处"} ] } }
```
- **自著**：`RoutineAuthor` 用便宜模型从 **人设 + 近期生活记忆（实际爱做什么）+ 可选人给种子** 生成/修订；定期（`routineReviseEvery`）或生活重心变化时自修订（慵懒的髭切会把练习排少、发呆排多）。`authoredBy` 记 provenance。

**(b) 被安排队列 AssignmentQueue（"被安排做什么"）** `life_sim_assignment`：
```
id, presetId, desc, dueDay?, dueBlock?, source(约定|审神者|近侍|主控), assignedBy, status, threadId?
```
来源：①对话/互聊约定（open_threads 带 due）②审神者直接吩咐 ③近侍/主控（多 bot）。

**(c) 每日计划 DailyPlan** `life_sim_plan`：blocks 带 `start/activity/location/source(routine|assigned|improvised)/assignedBy/status(pending|done|skipped|interrupted)`。唤醒时（`planRegenAt`）由 DailyPlanner 用便宜模型（或纯程序合并+轻润色）= 常规日程 + 到期 Assignment + 昨日摘要 + 人设 生成。

**与 roll 的关系**：EventRoller 先查"当前块计划在做什么"，事件默认执行计划、或偏离、或块间自由（`plan_adherence` 四态）。**计划是主干，事件是纹理**——大幅降漂移、让世界连贯。`next_delay` 可对齐块边界。
**重规划**：`plan_adherence=interrupted`（雨天取消练习/审神者临时安排/别的 agent 来访）→ `DailyPlanner.replan` 从该点重生成，旧块标 interrupted/skipped。
**闭环价值**：真人「明天陪我做 X」/ 互聊「约了对练」→ Assignment → 明天真进模拟日程 → 下次「今天照你说的做了」。**日程把对话·事件·记忆串成因果链。**

**(e) 治理三层级（自己 → 近侍 → 审神者）**（和刀剑乱舞设定一致）：
- **① 自己**：自著常规日程、对被安排项按人设接受/协商/婉拒。
- **② 近侍 bot**（`kinjiPreset`，可轮值）：本丸总管，**审核并安排他人日程**——协调跨 bot 冲突、编排共同活动、汇总"本日の予定"、把 orchestrator 的跨 agent 意图落成各 agent 的 Assignment。**近侍 = 高维主控在世界内的自然化身**。
- **③ 审神者**：终审权，经 console 检查/修改任何日程、换近侍。
- 审批流：自著/自排 → 近侍审核编排（跨 bot）→ 审神者随时覆盖；冲突/越权近侍先裁、审神者兜底。

### 5.4 WorldContext + 动态事件类型 + 连续性 + 出本丸许可

**(a) WorldContext（本丸级，跨 agent 共享）** `life_sim_world`：
```jsonc
{ "clock":1719705600, "timeOfDay":"午后", "season":"初夏", "weather":"晴",
  "locations":["本丸·主屋","练习场","库房","庭院","檐下","厨房"],
  "externalLocations":["城下町","近所","遠征先·山道"], "updatedAt":1719... }
```
- weather 每 `weatherTickHours`（默认 6h）按**马尔可夫相邻表**推进（晴→多云→雨→阴），所有 agent 共享同一片天；`weatherSource: internal|api`，`clockScale` 可配。

**(b) 事件类型注册表（数据驱动、可增减、热重载）** `event-types.json`：条目 `{type, conditions{timeOfDay?/weather?/season?/location?/requiresOtherPresent?}, weight}`；roll 前按 WorldContext 筛出可发生集喂模型。

**(c) 时空连续性约束**（防"5 分钟前在纽约、一小时后到日本"）：时钟单调；地点合法（本丸内 ∪ 已许可外部）；移动花合理时间（不瞬移）；天气仅相邻转移（禁跳变）；季节随真实日期。约束只管"物理可能"，不限"人设驱动选择"。

**(d) 出本丸 = 许可制 + 外部世界书（不硬禁）**：预授权 `externalLocations` 白名单可直接去；白名单外 → bot 产出"想去做 X"诉求经 want_to_share 请审神者许可，获批才发生（MVP 先只用白名单）；外部世界书（§5.5 的一册）补外部 canon；出/归花路程时间。硬禁止只剩**新命名角色/重大伤亡**。

### 5.5 世界书：可扩展多兼容 canon 层（加类目不改架构）

- **(a) 类目开放**：`category` 自由字符串，代码永不硬编码类目常量，config 里按字符串引用；加类目 = 加条目、零代码。
- **(b) 前向兼容 schema**：核心字段沿用 worldbook + 开放扩展袋 `ext{}`（任意 KV）；loader 宽容，未知字段保留不报错。未来加 `region/rating/validSeason/…` 塞 `ext`，老代码不受影响。
- **(c) 多格式 adapter**：`st`（复用 `convertStWorldbook`）/`koishi`/`typed` 归一到内部超集；新格式 = 加 adapter。
- **(d) 按用途分册 + 稳定查询**：`worldbooks:[{path,purpose}]`（persona/external/reference/未来任意）；只走 `query({purpose?,categories?,scanBuffer})→entries`。加册、加类目、加 ext 维度对 sim 全透明。
- 定性为对现有 worldbook 插件的**小幅前向兼容增强**（老条目不带 `ext` 也照跑）。

### 5.6 反思 Reflect（斯坦福）

- **触发**：累积重要度（近期事件 importance 之和过阈，仿斯坦福>150 的轻量版）或夜间固定跑。
- **过程**：取最近若干条 → LLM 抽"几个最值得追问的高层问题" → 每问按 key/相关记忆检索 → 合成"高层洞见"（关系变化、自我认知、习惯发现）→ 写回**长期**（可再被反思，成树）。
- **产物用途**：喂给 RoutineAuthor（习惯养成）、关系档案（关系演化）、下次 roll（自我一致）。

### 5.7 想法缓存区 ThoughtBuffer（心事簿）

`life_sim_thought`：`{id,presetId,content,target(审神者|presetId|self),origin,urgency,status(pending|surfaced|dropped|merged),relatedThreadId,createdAt,revisedAt}`。
- **存**：`want_to_share=later` 或"想说但不急"的念头入库（**取代 later+定时重查**：parking，不排闹钟）。
- **调**：下次和 target 接触时（真人来聊/经 ChatProxy 遇到那 bot），相关 pending 注入 `{pending_thoughts}` → "对了，之前一直想跟你说…"。
- **改**：更新/合并/作废（事已自解则 drop）；夜间随沉淀整理陈旧项。
- **分界**：livingmemory=过去事实、ThoughtBuffer=想说未说的意图、open_threads=进行中处境；说出即 `surfaced` 并沉淀成记忆。

### 5.8 主动外联（人设主导 + 两道可开关硬熔断）

> 研究警报：HBS 审计 1200 次告别 43% 用情感操控被骂粘人。**主动性优化参与度 = 必然读作粘人。**

- **决策交给人设**：`want_to_share.decision`（now/later/no）由角色性格决定想不想说、何时说、没回是否再发（沉默状态当输入喂进去）；无 importance 门槛。system 禁六类操控话术，draft 只分享真事。`later` → ThoughtBuffer。
- **两道可开关硬熔断**（默认开，可关成全人设驱动）：① `quietHoursEnabled` 安静时段不发；② `dailyCapEnabled` 每日上限（防写崩人设刷屏）。它们不是重要度门槛，是"不半夜发、一天别发二十条"的物理礼貌。
- **发送**：经 relay 吃其限流/白名单/防环；`proactiveEnabled` 可一键关（退化为纯被动浮现，最安全）。

### 5.9 持久化调度（重启不失忆）

`life_sim_task`：`{id,presetId,fireAt,type(roll|consolidate|prune|plan|reflect),status,payload}`。`ready` 扫过期→单次补跑（不雪崩，过老顺延）+ 未来的 `setTimeout`；roll 完插下条（自循环）；`dispose` 清内存 timer、DB 保留；同 preset 单 running roll 锁。

### 5.10 夜间「睡觉」

到 `sleepHour`：短→长沉淀（§4.3）+ 标记计划 done/skipped、未完成 assigned 顺延/放弃 + 生成次日计划 + 整理 ThoughtBuffer + 反思。

## 6. 可行性与坑

### 6.1 成本账
- 醒着约 14h，事件间隔多 40–120min → **8–20 次 roll/天/agent**；单次输入 ~1.5–2.5k、输出 ~300–500 token。
- 便宜模型：默认**本地 ollama 免费**；云端 gemini-flash/siliconflow 约每天几分钱/agent。夜间沉淀 1 次 ~3–5k。
- 省钱：静态前缀吃 prompt cache；事件驱动非轮询（省 10–20×）；FrugalGPT 级联（本地兜底，低置信升云端）；`dailyRollCap`。多 agent 全开云端也就每天一两毛，本地近乎免费。

### 6.2 调度可靠性
- koishi 定时器重启即丢、无 cron → §5.9 DB 落库+启动扫描是**必须**。过期任务单次补跑，不堆积雪崩。

### 6.3 漂移与失控（半自由叙事如何不崩）
schema 约束 + life-state 锚 + 每次重注人设 canon + worldbook 硬护栏 + 动态类型集/硬禁止清单 + 计划当主干 + dry-run（只 log 不写库调参）+（后置）canon 校验不合规重 roll。

### 6.4 与真人聊天的衔接
sim-memory 多变量**直接注入**，和 livingmemory 并排 → 自然不突兀。关键不变量：sim 记忆与真人记忆**格式同构**。

### 6.5 主动外联的雷区
见 §5.8：模拟(常跑)与外联(极少)分离 + 人设主导 + 禁操控话术 + 两道可开关熔断。

### 6.6 多 agent 互聊
见 §3.3：ChatProxy 便宜模型内部模拟(不跑主模型窗口) + 轮数帽 + TERMINATE + 哈希 + 防 echoing + `botChatDailyCap`。

## 7. 额外功能（锦上添花）+ 优先级

| 功能 | 价值 | 触发 | 挂到核心哪里 | 优先级 |
|---|---|---|---|---|
| **心情/状态波动** | 情绪连续性，染色后续事件与语气 | 每事件 mood 影响 → 更新 life-state.mood → 回喂下次 roll & 对话语气 | life-state + 短期记忆 | **P1**（字段已在，先轻量） |
| **bot 主动分享今天的事** | 温度、主动性 | 人设 want_to_share=now + 熔断；或夜间"今日小结"式分享 | ProactiveBridge | **P1**（核心已含最小版） |
| **周报/日记生成** | 用户可读产物、有存在感 | 夜间/每周主模型总结当期 → 第一人称日记/周报 | 读短期/事件流 → 写日记条目（可发用户/存档） | **P2** |
| **突发事件** | 打破常规、惊喜 | 低概率 roll 注入计划外（来客/器物损坏/天候突变/出阵警报） | EventRegistry「突发」类 → 触发 replan | **P2** |
| **节日/季节事件** | 世界有日历感 | 真实日期 → 本丸节庆（正月/花见/七夕） | WorldContext + 节庆日历 → 特殊事件 + 共同 Assignment | **P2** |
| **刀剑男士关系演化** | 活的关系，亲密度/基调随交互漂移 | 每次 ChatProxy 更新关系档案 + 关系里程碑事件 | 每对关系档案 + 反思 | **P2→P3**（需多 agent） |
| **梦境事件** | 夜间反思的叙事化，趣味（呼应 dream 双关） | 夜间沉淀时生成一段"梦" | 复用反思 + 夜间沉淀 | **P3** |
| **习惯养成** | 常规日程从行为演化 | 反复做的活动 → 反思发现 → 修改自著 routine | RoutineAuthor 自修订 | **P3** |
| **纪念日/回忆** | 旧事在周年浮现 | 日期匹配长期重要事件 | 长期记忆 `date` key 直取 | **P3** |

（这些都**只挂在既有核心接口上**：事件走 EventRegistry/roll、记忆走长短期两层、日程走 Planner、主动走 ProactiveBridge——不新增架构。）

## 9. 分阶段 MVP

### Phase 1 — 单 agent 自循环（先做，最划算）
独立验证"不聊天世界也在长 + 可引用一致"，80% 价值在此：
1. **持久化 Scheduler**（DB+ready 扫描+dispose）——硬地基。
2. **IdleWatcher**（订阅 message_collect）。
3. **WorldContext + EventRegistry**（含连续性约束）。
4. **RoutineAuthor + DailyPlanner + DailyPlan**（bot 自著常规日程 → 每日计划当主干；唤醒生成；重规划/对话安排留 P2）。
5. **审神者查看/修改日程**（console/指令）。
6. **EventRoller（Act）**（便宜模型 + schema + life-state 锚 + worldbook + 当前日程块 + 动态类型集 + list-then-roll；fallback 兜底）。
7. **ShortTermMemory**（直取式短期工作集 + 各表）。
8. **NightConsolidator**（**短→长沉淀**：聚类 keep/update/merge/archive 进 LongTermMemory；标记计划 + 生成次日计划）。
9. **ThoughtBuffer**（对审神者：store + 下次来聊 surface + revise）。
10. **心情轻量**（mood 字段随事件波动）。
11. **最小 ProactiveBridge**（人设 want_to_share 主导 + 可开关熔断；later→ThoughtBuffer）。
12. **PromptProvider** 注入 `{recent_life}/{life_state}/{today_plan}/{pending_thoughts}`。
13. dry-run + debug + 本地模型默认。

**验收**：开一两天后，`life_sim_event` 有贴合日程主干、随天气/时段变化的连续事件流；长期库有夜间沉淀的条目；下次聊天它能自然、**一致**地提起这两天做的事（你追问对得上）；主动消息符合人设节奏且不粘人。

### Phase 2 — 打磨与深化
- **反思 Reflector**（累积重要度触发，写回长期）。
- **长期记忆压缩/遗忘**（dream 式 + 引用计数）+ 可选给长期建 embedding 做联想深取。
- **周报/日记生成**、**突发事件**、**节日/季节事件**。
- 日程闭环：AssignmentQueue 录入「对话约定/审神者吩咐」+ 重规划；出本丸"请求许可"流。
- canon 校验（不合规重 roll）。

### Phase 3 — 多 agent
- **ChatProxy**（便宜模型多轮互聊 → transcript + 每对关系档案 + 双方各若干条记忆；防跑飞全套）。
- **每对关系档案**（key 直取）+ **关系演化**。
- **近侍协调 + Orchestrator**（谁与谁交互、跨 bot 日程编排）。
- `visibleVia: relay` 围观升级。

## 10. 风险登记

| 风险 | 等级 | 缓解 |
|---|---|---|
| 靠向量抓低价值日常、抓不到 | 高（已规避） | **记忆改直取式两层**：recency 短期直取 + key 长期直取，向量仅可选联想（§4.4） |
| 重启丢闹钟、本丸停摆 | 高 | DB 落库 + ready 扫描补跑（§5.9）——P1 第一件事 |
| 便宜模型 roll 崩人设 | 中高 | schema + life-state 锚 + worldbook + 动态类型集/硬禁止 + 计划主干 + dry-run（§6.3） |
| 主动消息读作粘人 | 中高 | 人设主导 want_to_share（含没回是否再发）+ 禁操控话术 + 两道可开关熔断（§5.8） |
| 世界跳变（瞬移/天气突变/乱跑外地） | 中 | 时空连续性约束 + 出本丸许可制（§5.4c/d） |
| 日程太死板、像打卡 | 中 | bot 自著日程 + 计划只当主干、事件可偏离/打断/自由；重规划（§5.3） |
| 与 livingmemory 打架 | 中 | 分域：sim 不写它向量、只借鉴 dream 思路；重要生活事件靠对话自然回流（§4.6） |
| 多 agent 互聊跑飞/成本爆炸 | 高（P3） | ChatProxy 便宜内部模拟 + 轮数帽 + TERMINATE + 哈希 + 防 echoing（§3.3/6.6） |
| 近侍越权 / 抢资源 | 中（P3） | 三层级：近侍先裁、审神者终审覆盖（§5.3e） |

## 11. 参考（人机恋 / agent 研究）

- **Generative Agents（Park 2023）**：memory stream + 检索三因子（recency·importance·relevance，等权 min-max 归一）+ reflection（重要度和>150 触发，100 条→3 问→每问 5 洞见，成树）+ top-down 规划（日→时→5-15min，反应即重规划）+ agent 对话（共处→取关系记忆→轮替至结束→记回流）。**本设计把三因子拆开分工、把空间共处换成近侍协调。**
- **AI Town（a16z）**：recency 衰减 0.99/h，importance 0–9，等权。
- **Letta sleep-time**：只读 primary + 后台 rewriter；分层 cadence（session 去重/session-end 轻整理/weekly 重组）；reference-count 比 age 更重要；别过度归并丢粒度。
- **Mem0**：ADD/UPDATE/DELETE/NOOP 冲突解决。
- **ComPeer**：事件检测 + 晨间反思 + 随机阈值门 + 3h 无回复抑制。
- **Nomi/Kindroid**：闲置触发 + 指数退避 + 硬安静时段 22:00–08:00 + "基于真在想/做的事"非脚本。
- **HBS 告别审计**：43% 情感操控 → 主动外联最大雷区。
- **Ian Bicking**：list-then-roll（LLM 列表→程序抽样）；分层世界状态。
- **AutoGen**：MaxMessage/TERMINATE/TokenUsage 终止；echoing 论文：每轮重注角色卡降 40–60%。
- **FrugalGPT**：便宜优先级联，最高 98% 降本。
- **livingmemory（本仓依赖）**：`living_memory_entry`（type/status/importance0-1/embedding）；`runDream` 两阶段聚类 keep/update/merge/archive；vector+importance 检索。**sim-memory 复用其整理思路、各管各域。**
